║                    Как это работает на самом деле?                            ║
║                                                                                ║
╚════════════════════════════════════════════════════════════════════════════════╝


❌ ВЫ ПОДУМАЛИ:
═══════════════════════════════════════════════════════════════════════════════

enable_upload: true
└─ Сразу загружается, не нужно переносить в to_transcribe_folder

enable_upload: false
└─ Нужно вручную переносить в to_transcribe_folder


✅ НА САМОМ ДЕЛЕ:
═══════════════════════════════════════════════════════════════════════════════

enable_upload: true
├─ Переименование: ✅ ВСЕГДА (в watch_folder)
├─ Перенос в to_transcribe_folder: ✅ ОБЯЗАТЕЛЬНО (пользователь)
└─ Загрузка на сервер: ✅ ЕСЛИ перенесли

enable_upload: false
├─ Переименование: ✅ ВСЕГДА (в watch_folder)
├─ Перенос в to_transcribe_folder: ❌ НЕ НУЖНО
└─ Загрузка на сервер: ❌ НИКОГДА


🎯 ОТВЕТЫ НА ВАШИ ВОПРОСЫ:
═══════════════════════════════════════════════════════════════════════════════

Вопрос 1: Если enable_upload: true, то не нужно переносить?
Ответ: НЕПРАВИЛЬНО! Вам ВСЕГДА нужно переносить в to_transcribe_folder!

Вопрос 2: Сразу будет запущена транскрипция?
Ответ: НЕТ! Только после переноса файла в to_transcribe_folder!

Вопрос 3: Если enable_upload: false, то нужно вручную переносить?
Ответ: НЕТ! Переносить не нужно, приложение не загружает в любом случае!


📊 ТАБЛИЦА:
═══════════════════════════════════════════════════════════════════════════════

enable_upload │ Переносить │ Загружается │ Транскрипция
──────────────┼────────────┼─────────────┼──────────────
true          │ ✅ ДА      │ ✅ ДА       │ ✅ ДА
true          │ ❌ НЕТ     │ ❌ НЕТ      │ ❌ НЕТ
false         │ ✅ ДА      │ ❌ НЕТ      │ ❌ НЕТ
false         │ ❌ НЕТ     │ ❌ НЕТ      │ ❌ НЕТ


🔄 ПРАВИЛЬНЫЙ ПРОЦЕСС:
═══════════════════════════════════════════════════════════════════════════════

ЕСЛИ enable_upload: true (хотите транскрипцию)
1. Видео появляется в watch_folder
2. Приложение переименовывает его
3. ВЫ ПЕРЕНОСИТЕ в to_transcribe_folder (ОБЯЗАТЕЛЬНО!)
4. Приложение загружает на сервер
5. Приложение обрабатывает транскрипцию
6. Результаты публикуются

ЕСЛИ enable_upload: false (только переименование)
1. Видео появляется в watch_folder
2. Приложение переименовывает его
3. НЕ ПЕРЕНОСИТЕ в to_transcribe_folder (не нужно)
4. Файл остается в watch_folder
5. Конец


💡 КЛЮЧЕВОЕ ОТЛИЧИЕ:
═══════════════════════════════════════════════════════════════════════════════

enable_upload контролирует ЧТО ПРОИСХОДИТ
└─ true = загружается на сервер
└─ false = не загружается

Двухэтапная архитектура контролирует КОГДА это происходит
└─ Этап 1 (watch_folder) = переименование (ВСЕГДА)
└─ Этап 2 (to_transcribe_folder) = загрузка (ЕСЛИ пользователь переносит)


🎯 ИТОГОВАЯ ЛОГИКА:
═══════════════════════════════════════════════════════════════════════════════

Переименование ВСЕГДА происходит в watch_folder
├─ Независимо от enable_upload
└─ Независимо от того, переносите ли вы файл

Загрузка происходит в to_transcribe_folder ТОЛЬКО если:
├─ enable_upload: true
└─ Вы переносите файл в to_transcribe_folder
